# Test vector provenance

Real committed compressed-asset blobs, copied read-only from sibling
workspace projects (not the sigil repo) to serve as round-trip gates. The
matching `.raw` files are the plain (decompressed) payload.

| File | Source path | Format | Compressed size | Decompressed size |
|---|---|---|---|---|
| `oozprimary.kos` / `.raw` | `s2disasm/collision/OOZ primary 16x16 collision index.kos` | Kosinski (plain) | 256 B | 768 B |
| `specstag.kos` / `.raw` | `s2disasm/art/kosinski/SpecStag.kos` | Kosinski (plain) | 816 B | 1492 B |
| `sand_particles.kosm` / `.raw` | `skdisasm/Levels/SOZ/KosinskiM Art/Sand Particles.bin` | Kosinski-Moduled | 194 B | 416 B |
| `ship_propeller.kosm` / `.raw` | `skdisasm/Levels/MHZ/KosinskiM Art/Ship Propeller.bin` | Kosinski-Moduled | 514 B | 864 B |
| `level_select_2p.eni` / `.raw` | `s2disasm/mappings/misc/Level Select 2P.eni` | Enigma | 82 B | 408 B |

Module size for both `.kosm` files is `$1000` (the S3K/SK default) —
verified during recon by decompressing both at that module size; output was
plausible-looking tile data of the expected size.

## `.raw` provenance: NOT generated by this crate's own decompressor

**Load-bearing detail**: the `.raw` files were NOT produced by the naive
raw-pointer `ClownLZSS::*Decompress` path (which has a real, confirmed bug —
see `src/shim.cpp`'s big comment on `DecompressorOutput::Copy` and
`std::copy`/`memmove` overlap semantics). They were produced by the
corrected `std::ostringstream`-based decompressor (the same code path this
crate's `decompress_*` functions use) and then **cross-checked against an
independent, unrelated Kosinski implementation**
(`programs/accurate-kosinski` elsewhere in this workspace — also written by
Clownacy, but a from-scratch C implementation with no code in common with
`clownlzss`) for every Kosinski/Kosinski-Moduled `.raw` file. All matched
exactly. This two-source cross-check is what gives confidence these `.raw`
files are the TRUE plaintext, not merely "whatever our own (initially
buggy) tooling produced."

## Compression goldens

`golden_*.bin` files were captured by a standalone driver linked directly
against the vendored `compressors/*.h` templates (constructing a
`ClownLZSS::CompressorOutput<unsigned char*>` explicitly and reading back
`.Tell()`, exactly like this crate's `src/shim.cpp` does — see its "Tell(),
not the caller's pointer" comment for why that indirection is required),
compressing `level_select_2p.raw` (408 bytes, word-even). These are
regression vectors: "byte-exact vs clownlzss@8055bd2", NOT byte-exact vs
Sega's own compressors.

## Saxman

No committed Saxman-compressed binary blob was found anywhere in this
workspace (see the crate's `tests/byte_exact.rs` Saxman test doc comment for
the search performed and the documented synthetic-fallback decision).

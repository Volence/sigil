# Sigil M1.C — T9.2: `.ATTRIBUTE` + `!name` escape + `while` Plan

> **For agentic workers:** REQUIRED SUB-SKILL: subagent-driven-development. TDD, asl-diff-gated.
> CI runs `cargo clippy --workspace --all-targets -- -D warnings`.

**Goal:** Three `__DEBUG__`-supporting features: `.ATTRIBUTE` macro-suffix synthesis, the `!name`
builtin escape (`!error`/`!align`), and the `while … endm` loop. Depends on T9.1.

## Verified asl semantics (probed 2026-07-04)
- **`.ATTRIBUTE`:** a macro invoked as `name.SUFFIX args` binds `.ATTRIBUTE` → `.SUFFIX` inside
  the body. `foo macro src / move.ATTRIBUTE src,d0 / endm`, then `foo.w d1` → `move.w d1,d0` =
  `30 01`; `foo.l d2` → `move.l d2,d0` = `20 02`. So the invocation's trailing `.b`/`.w`/`.l` is
  captured and text-substituted for `.ATTRIBUTE` in the body (works in mnemonics like
  `cmp.ATTRIBUTE`/`move.ATTRIBUTE` AND in strings like `%<.ATTRIBUTE src>`).
- **`while (cond) … endm`:** loops while `cond` is nonzero, re-evaluating each iteration (works
  with a `set` accumulator). **Terminator is `endm`** (NOT `endw` — asl errors "WHILE without
  ENDM"). `n set 0 / while (n<3) / dc.b n / n set n+1 / endm` → `00 01 02`.
- **`!name`:** `!error "msg"` / `!align N` force the AS builtin over a same-named user macro. In
  Core there are no shadowing user macros, so `!name` = strip the `!` and run the builtin
  directive `name`. `!error` aborts like `fatal`/`error`; `!align 2` = `align 2`. (`even` is NOT
  a valid asl directive — verified "unknown instruction" — so no `even` handling.)

## Files
- `crates/sigil-frontend-as/src/eval.rs` — (1) macro dispatch: when `head` is not a known macro
  but stripping a trailing `.b`/`.w`/`.l` yields a known macro, expand it binding `.ATTRIBUTE` →
  the suffix (text-substitute in the body, like a positional param — reuse T7's substitution).
  (2) `exec` block handler for `while`/`endm` (a bounded loop: eval the `(cond)` via `eval_all`
  each iteration, expand the body, cap iterations with the A5 non-convergence diagnostic). (3)
  `!name` handling in `dispatch`/`exec_one`: a leading `!` on a directive head strips to the
  builtin. Register `while` in `is_op_keyword`; keep `endm` as the shared terminator.

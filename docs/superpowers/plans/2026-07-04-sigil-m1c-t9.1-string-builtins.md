# Sigil M1.C — T9.1: Debug String Builtins + `!`=bitwise-or Plan

> **For agentic workers:** REQUIRED SUB-SKILL: subagent-driven-development. TDD, asl-diff-gated.
> CI runs `cargo clippy --workspace --all-targets -- -D warnings`.

**Goal:** Add the AS string builtins the `__DEBUG__` debugger uses — `substr`, `strlen`,
`strstr`, `val` — plus the `!`=bitwise-or infix operator. Foundation for T9.2/T9.3.

**Architecture:** These are FRONT-END builtins (§7.4 — strings never enter `sigil_ir::Expr`).
Build a small front-end string+int evaluator (mirroring T8b's `eval_float` structure): `substr`
returns a String; `strstr`/`strlen`/`val` consume strings and return `i64`; nesting supported
(`strlen(substr(...))`, `strstr(substr(...),">")`). Wire it into the expression fold where these
calls appear (if-conditions, `set`/`dc.b` operands inside debugger macros).

## Verified asl semantics (probed 2026-07-04, asl 1.42 Bld 212)
- `substr(str, pos, len)` → substring; **`len=0` means "from pos to end"**. `substr("hello",1,0)`
  = "ello", `substr("hello",1,2)` = "el". (0-based pos.)
- `strlen(str)` → length. `strlen("hello")`=5.
- `val(str)` → parse the string as an AS number: `val("$80")`=0x80, `val("144")`=144(=0x90).
  (Handles `$hex`, decimal, likely `%bin`/`h`-suffix — probe the forms the debugger passes.)
- `strstr(haystack, needle)` → **STANDARD**: 0-based index of first match, or **-1 if absent**.
  **D5 CORRECTION (verified):** the alleged "fails on last char" bug does NOT reproduce in this
  asl — `strstr("abc","c")`=2, `strstr("b>",">")`=1, `strstr("xab","ab")`=1, `strstr("abc","z")`
  =-1. Implement STANDARD strstr (no bug emulation); it reproduces the debugger bytes exactly
  (proof deferred to the T9.3 `%<…>` gate). See design doc D5.
- `!` = infix **bitwise-OR** (same as `|` → `BinOp::Or`): `((*)&1)!1` pattern. Precedence: probe
  asl (`3!4`, `1!2&3`) — likely the same tier as `|`.

## Files
- `crates/sigil-frontend-as/src/expr.rs` — map `!` (infix) → `BinOp::Or` in `infix_bp`. (The
  lexer already has `Punct` for `!`? check — `!=` exists; a bare `!` infix may need a Punct. The
  `!name` escape is T9.2, at statement level, distinct from infix `!`.)
- `crates/sigil-frontend-as/src/eval.rs` — a front-end string/int builtin evaluator
  (`eval_str`/`eval_strbuiltin`) recognizing `substr`/`strstr`/`strlen`/`val` + `Tok::Str`;
  wired into the operand/condition fold path. Keep strings out of `sigil_ir::Expr`.

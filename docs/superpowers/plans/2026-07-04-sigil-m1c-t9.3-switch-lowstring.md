**Goal:** Add the final debug primitives — `lowstring()` builtin + `switch/case/elsecase/endcase`
string dispatch — so the debugger's `%<…>` format-bytecode machinery (already written in terms of
`while`/`strstr`/`substr`/`val`/`switch`/`lowstring`, all now available) assembles byte-exact.

## Verified asl semantics (probed 2026-07-04)
- `lowstring(str)` → lowercased string. `lowstring("HeX")` = "hex". (A string builtin like `substr`
  — returns a String; add to T9.1's string evaluator.)
- `switch <str-expr>` / `case "s1"` / body1 / `case "s2"` / body2 / `elsecase` / default /
  `endcase` → assembles ONLY the body of the first `case` whose literal string equals the switch
  value; `elsecase` is the default. Verified: `switch lowstring("HeX") / case "hex" / dc.b $80 /
  case "dec" / dc.b $90 / elsecase / dc.b $FF / endcase` → `80`.

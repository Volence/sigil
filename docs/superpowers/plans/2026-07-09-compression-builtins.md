## Frozen rulings

- **CR1 (CONFIRMED) — builtins are OPTIMAL compressors, not Sega-accurate.** Byte-exact
  stock-ROM rebuilds keep using `embed()` of original blobs. Sega-accurate encoders are
  OUT (deferred; note: clownnemesis has an `accurate=1` Shannon-Fano mode a follow-up
  could expose as a named arg for ~zero cost — recorded, not built).
- **CR2 (AMENDED: license + shim) — vendor Clownacy upstream, TWO crates.**
  - `sigil-clownlzss-sys`: clownlzss @ `8055bd2e2d258dab604ea26ded9e42144562db54`.
    Reality check: the optimal-parse core (`compressors/clownlzss.c`) is C89, but EVERY
    format compressor/decompressor is a **C++20-template header with no C entry point** —
    the crate hand-writes ONE small C++ shim (`shim.cpp`, `extern "C"` per-format
    functions), compiled via `cc::Build::cpp(true).std("c++20")`. Shim output contract
    mirrors salvador: max-compressed-size query + caller-allocated buffer, with the shim
    instantiating templates against a raw `unsigned char*` (random-access iterator — this
    is REQUIRED, not a convenience: Saxman/Rocket write seek-back-patched headers, which
    a `back_inserter` cannot support).
  - `sigil-clownnemesis-sys`: clownnemesis @ `7abcddc7e220f33c1e03ccaecb46a0c0323182c8`
    + its `clowncommon` submodule @ `37d1efd` (single header, required by compress.c).
    Plain C89, callback-based streaming API; smoke-tested clean with plain gcc. **The
    load-bearing wrapper contract (recon-verified by live repro): the input callback MUST
    rewind to the start on EOF** — the encoder is two-pass; a naive exhaust-once reader
    silently emits truncated "successful" output.
  - **License: ISC-style Clownacy text, NOT 0BSD** (the handoff's research overstated;
    no SPDX tags upstream). Same zero-obligation permissiveness — vendor-clean — but the
    crates' `license` metadata and vendored LICENCE files must say what's true (ISC
    text verbatim), not copy salvador's labeling. s2disasm's local snapshot (zlib-era,
    Saxman-only, malloc-returning C API) is confirmed dead-end — vendor upstream only.
- **CR3 (CONFIRMED, scope sharpened) — tiers:**
  - **Tier 1 (FIRST): `s4lz()`** — pure-Rust byte-exact port of `aeon/tools/s4lz.py` v3
    incl. header, optimal-DP parse, hash-chain matcher WITH its exact heuristics
    (early-exit ≥32 words), dictionary mode (the real ojz consumer shape), tile_delta.
    Surface: `s4lz(data)`, `s4lz(data, dict: d)` (word-even dict), `s4lz(data,
    tile_delta: true)`; `dict`+`tile_delta` = loud error (Python ValueError parity).
    v1 legacy = decode-only in Python, never emitted → NOT implemented. Gate: committed
    input/output vector pairs generated by the REAL s4lz.py (provenance README), covering
    the survey's shape list (744-byte selftest payload plain+dict, real-asset 768-byte
    blocks with 768/1536/2304-byte dicts, empty/odd/510-vs-512-offset/ext-count edges,
    tile_delta). The ojz K-sweep/dict-SELECTION stays the caller's concern
    (`ojz_block_gen.py`) — the builtin is `compress()`, not the packer.
  - **Tier 2: classic formats via CR2**, one commit per format, each independently
    gated; priority order: `kosinski()`, `kosinski_m(data, module_size: N=$1000)`
    (module_size > $1000 = loud error — the 12-bit moduled header can't represent it;
    upstream merely warns), then `kosplus()`, `saxman(data, header: bool=true)` (both
    upstream variants exist), `enigma()`, `nemesis()`, and — free, same template shape —
    `comper()`, `rocket()`. Gates per format: (a) compression golden = committed output
    of the vendored code itself, captured by a vendor-time driver, documented as
    "byte-exact vs clownlzss@8055bd2 / clownnemesis@7abcddc", NOT vs Sega; (b)
    round-trip sanity vs real committed workspace blobs (s2disasm/skdisasm .kos/.nem/.eni
    farm): vendored-decompress(original) → recompress → vendored-decompress → equal
    plain bytes.
  - **Tier 3 (design note only, no code):** decompression builtins (`kosinski_dec()` …)
    — aeon's `ojz_common.py` build-time Kosinski DEcompressor (legacy asset migration)
    is the proven use case; the vendored decompressors make this cheap when a real
    migration consumer appears. Deferred.
- **CR4 (CONFIRMED) — raw format streams.** Classic builtins emit the raw stream, no
  aeon 4-byte wrapper (that was zx0()-specific aeon convention). `s4lz()` emits the v3
  header because it IS the format. Future wrapping composes in `.emp`.
- **CR5 (CONFIRMED, constraints enumerated) — diagnostics.** zx0 taxonomy per builtin
  (`[kosinski.symbolic]`, `[s4lz.too-large]`, …), all loud, never panic. Format
  constraint table (from the vendored code): s4lz data ≤ u16 size field, dict word-even;
  nemesis input multiple of $20 (tile granularity), ≤ 32767 tiles (15-bit header field);
  enigma + comper input word-even; kosinski_m module_size ≤ $1000; saxman-with-header
  compressed size ≤ u16. Vendored decompressors return void/unchecked → the round-trip
  test harness bounds-checks on the Rust side.

# Sigil M0.5 — Procedural-EA Spike (`sigil-isa::m68k` seed)

**Date:** 2026-07-03
**Status:** Approved (brainstorming), ready for writing-plans
**Track:** Sigil Core backend/linker (M1 lead-in). *Not* the Spec-2 `.emp` surface language.
**Predecessor:** M0 complete & merged (`sigil` master @ `be86754`): byte-exact Z80 driver regions A + B.

## 1. Goal & risk retired

Prove that a **procedural EA / extension-word encoder** for the base 68000 produces
byte-identical output to `asl` (asl 1.42, `aeon/tools/asl`), *before* committing to the
full `sigil-backend-m68k` in M1.

The design risk being retired is the **procedural EA machinery** (the §5.2 escape hatch) —
not the fixed-field table (moveq/addq/branches are regular, low-risk, table-expressible).
`MOVE` is the stress vehicle: a single mnemonic exercises nearly every addressing mode in
**both** field positions, so it directly proves the #1 byte hazard (§5.5 `MOVE` dest-EA
mode/register field swap) and the whole EA-mode matrix at once.

**Success:** a committed golden-vector corpus of `MOVE` across the EA-mode matrix, every
form byte-matching `asl`, with the dest-swap exercised in both operand positions and the
`(d8,An,Xn)` brief extension word proven.

This is framing **B** from brainstorming: the spike is the **first real vertical slice** of
the encoder with committed golden vectors — M1 *grows* the corpus, it does not restart.

## 2. Where the code lives (follows the M0 precedent)

The M0 precedent is unambiguous and is the model:
- `sigil-isa` holds the real encoder (`z80` module), **zero workspace deps**,
  extraction-ready per the ISA-sharing decision (`SIGIL_ORACLE_ISA_SHARING.md`, 2026-07-01).
- `sigil-backend-z80` is a *thin* `Backend`-trait adapter (`sigil-ir` + `sigil-isa` +
  `sigil-span`) that re-exports the isa vocabulary so the AS front-end never takes a direct
  `sigil-isa` edge.

Therefore:
- **`crates/sigil-isa/src/m68k.rs`** — the real encoder module, mirroring `z80.rs`.
  **Zero workspace deps** (must not break crate-graph guard rule (a)). Dual-facet
  (decode/disasm) and the disassembler are **noted-but-deferred**, exactly as full Z80
  disassembly was deferred in M0.
- **No `sigil-backend-m68k` crate in the spike.** The spike is a pure encoder + golden-vector
  slice, tested directly — as M0's Z80 encoder was proven in `sigil-isa` independent of the
  thin backend adapter. The backend adapter is M1 proper.
- No `crate_graph.rs` edits needed: rule (a) already asserts `sigil-isa` has zero workspace
  deps; the new module preserves that.

*(Note: the handoff's "backend depends on sigil-ir only" restates spec §9.1's crate graph;
the operative precedent from M0 is that the encoder lives in `sigil-isa` and the backend is
a thin adapter over it. The spike follows the precedent.)*

## 3. Operand / instruction model & the width-selection boundary

Mirror the Z80 model:

```rust
pub enum Mnemonic { Move, /* … grown in M1 */ }
pub enum Size { B, W, L }            // B included for completeness; MOVE slice uses W/L
pub enum Xn { D(u8), A(u8) }         // index register for the brief extension word

pub enum Operand {
    Dn(u8),                          // Dn
    An(u8),                          // An
    Ind(u8),                         // (An)
    PostInc(u8),                     // (An)+
    PreDec(u8),                      // -(An)
    Disp16An(i16, u8),               // (d16,An)
    Disp8AnXn { d: i8, an: u8, xn: Xn, long: bool }, // (d8,An,Xn[.w|.l]) → brief ext word
    AbsW(i16),                       // (xxx).w
    AbsL(i32),                       // (xxx).l
    Pcd16(i16),                      // (d16,PC)
    Imm(i32),                        // #imm
}

pub struct Instruction { pub mnemonic: Mnemonic, pub size: Size, pub ops: Vec<Operand> }

pub fn encode(inst: &Instruction) -> Result<Vec<u8>, IsaError>;
```

Each EA is an **already-resolved form** — the encoder is a pure function over concrete
integers (§5.3 contract: "the backend receives resolved integers").

**Critical boundary decision.** `abs.w` vs `abs.l` vs `(d16,PC)` **selection** (§5.6) is
**out of spike scope**. The spike proves each form *encodes* correctly by making the operand
carry its explicit form (`AbsW` vs `AbsL` vs `Pcd16`), driven by the `.w`/`.l` suffix in the
asl snippet. The *selection policy* (which form AS picks for a symbolic operand) is a named
M1 deliverable, deferred. This keeps the spike a pure-encoder test and matches §5.3.

**The one shared EA function.** `encode_ea(op, field) -> (mode3, reg3, ext_words)` is used for
**both** source and dest. The MOVE word places:
- **dest** EA as `register:mode` into bits 11–6, and
- **source** EA as `mode:register` into bits 5–0.

That single asymmetry *is* the §5.5 MOVE hazard, and is proven by the dest-position vectors —
not a second EA encoder.

## 4. Golden-vector infrastructure (verbatim Z80 pattern)

- **`crates/sigil-isa/tests/corpus_m68k/mod.rs`** — canonical
  `corpus_m68k() -> Vec<(&'static str, Instruction)>`, the single source of truth, shared via
  `#[path]` by the generator and both tests. (Cargo does not compile a `tests/<name>/mod.rs`
  as its own integration binary — the same trick M0 used for `tests/corpus/mod.rs`.)
- **`src/bin/gen_m68k_vectors.rs`** — manual developer tool (NOT run in CI). For each snippet
  it assembles with `aeon/tools/asl -cpu 68000 -q -L -U` + `p2bin`, extracts the exact bytes,
  and (over)writes `tests/m68k_golden_vectors.txt` as `<snippet> => <space-separated
  uppercase hex>` in `corpus_m68k()` order. `AEON_DIR` env override, defaulting to
  `/home/volence/sonic_hacks/aeon`. Snippet wrapper:
  `        cpu 68000\n        org 0\n        <snippet>\n`.
- **`tests/m68k_golden_vectors.txt`** — committed; CI reads it and never runs asl.

**The corpus (~18 vectors), all concrete literals — no symbols, so `p2bin` yields exact bytes:**

Baseline reg↔reg:
- `move.w d1,d0`
- `move.l a1,d0`

Source-mode sweep into `d0` (proves source EA in bits 5–0):
- `move.w (a1),d0`
- `move.w (a1)+,d0`
- `move.w -(a1),d0`
- `move.w (4,a1),d0`
- `move.w (6,a1,d2.w),d0`      *(brief extension word, index D, `.w`)*
- `move.w ($1234).w,d0`
- `move.w ($12345678).l,d0`
- `move.w (8,pc),d0`
- `move.w #$1234,d0`

Dest-mode sweep from `d1` (proves the dest-EA mode/register swap in bits 11–6):
- `move.w d1,(a0)`
- `move.w d1,(a0)+`
- `move.w d1,-(a0)`
- `move.w d1,(4,a0)`
- `move.w d1,($1234).w`
- `move.w d1,($12345678).l`

Size + extension-word `long` coverage:
- `move.l (2,a3,a4.l),d0`      *(`.l` size, index A, `long` flag set)*

*(Exact snippet strings are settled at implementation time by what `asl` accepts verbatim;
each string must match its golden-file key character-for-character. The list above is the
representative set; the count/spelling may shift slightly to satisfy asl's syntax.)*

## 5. Test wiring & acceptance

- **`tests/m68k_golden.rs`** — reads the committed file, asserts it is well-formed and
  non-empty (mirror of `z80_golden.rs`; may reuse `golden_common::parse_golden`).
- **`tests/completeness_m68k.rs`** — the acceptance gate: for every `corpus_m68k()` entry,
  assert `encode(inst) == golden[snippet]`, byte-for-byte.
- Green bars: workspace `cargo test` and `cargo clippy --workspace --all-targets -- -D warnings`.

**Spike acceptance criteria:**
1. Every corpus form encodes byte-identical to `asl`.
2. Dest-EA mode/register swap proven (dest-position vectors pass).
3. `(d8,An,Xn)` brief extension word proven for both index kinds and the `long` flag.
4. `AbsW` / `AbsL` / `Pcd16` are distinct forms and each encodes correctly.
5. Encoder is a pure function of resolved integers; `sigil-isa` retains zero workspace deps.

**Explicitly deferred to M1 (out of scope here):**
- `abs.w`/`abs.l`/`(d16,PC)` width *selection policy* (§5.6).
- Fixed-field hazards: moveq/addq/subq, `bra.s`/`bra.w`, `DBcc`, `MOVEM` `-(An)` mask reversal (§5.5/§5.7).
- Relaxation engine (§5.4) and linker fixups (§6).
- The thin `sigil-backend-m68k` adapter crate.
- The disassembler / dual-facet decode side (ISA-sharing Tier 3).

## 6. Non-goals / guardrails

- No AS-quirk logic in `sigil-isa` — the encoder receives resolved integers only; the
  quarantine (§9.1) stays intact.
- No peephole / mnemonic substitution.
- Do not reintroduce the removed duplicate 68k sound tables (aeon build hazard, per handoff).

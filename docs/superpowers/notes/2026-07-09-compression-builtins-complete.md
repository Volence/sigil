## The review cycle earned its keep (worth reading)

The T2a quality review found a **live Critical**: the moduled compress path had no shim-side
bounds check and an under-scaled Rust bound — reproduced as a real heap overflow
(`double free or corruption`, SIGABRT) on 32KB of incompressible input at module_size $1000.
The fix (`f8bd84f`) closed it defense-in-depth (shim capacity protocol — the shim can no
longer overrun regardless of caller; module-count-scaled bound + exact-size retry) and
surfaced two ADJACENT real bugs, now typed errors: `module_size 0` → division-by-zero UB in
the vendored wrapper, and quotient-over-15 inputs → the 16-bit moduled header silently
truncates upstream into a CORRUPT stream (`[<fmt>.data-too-large]`, enforce
`len ≤ 16·module_size − 1`). RED evidence is exemplary: the first repro attempt was "85
bytes too compressible" — the committed regression uses a B(256,2) de Bruijn prefix
(provably zero LZSS matches). Goldens confirmed the shim rewrite byte-identical.

Also caught earlier at the shim boundary (T2a implementation, not review): upstream's
`DecompressorOutput::Copy` raw-pointer path lowers to `memmove`, breaking LZSS
self-overlapping copies — routed via the `ostringstream` sink and cross-checked against the
independent `accurate-kosinski` decompressor.

## Design decisions

### D1 — module structure (dma_queue.emp)

`module engine.dma_queue in dma_queue`. Procs in twin order: Init_DMA_Queue,
QueueDMA_Critical, QueueDMA_Important, QueueDMA_Deferrable,
[QueueDMATransfer], Process_DMA_Critical, Process_DMA_Important,
Process_DMA_Deferrable, Drain_Budgeted_Queue.

**The QueueDMATransfer fallthrough**: the twin's QueueDMA_Deferrable falls
through into QueueDMATransfer (no branch — byte-mandatory, a bra.s to the
next instruction is unencodable). Two candidate .emp shapes:
- (a) four separate procs, Deferrable's body falls off its end;
- (b) the transfer core as `export .transfer:` INSIDE QueueDMA_Deferrable's
  body; Critical/Important tail-branch to `QueueDMA_Deferrable.transfer`.
Chooser: whichever the contract/out-verify checkers accept without
weakening a contract; tie → (b) (the fallthrough is explicit, the shared
core is visibly the tail of Deferrable). → **PROBE P7. RESOLVED: shape (b)
by the tie rule — BOTH shapes lower clean (P7a fall-off-end is NOT
rejected: the body's CCR-writing moves credit the carry out, so the
checker never sees the adjacency dependence). Shape (b) is taken on
explicitness: the shared core lives INSIDE QueueDMA_Deferrable's body
(`export .transfer:` → symbol `QueueDMA_Deferrable.transfer`), so the
byte-mandatory fallthrough is a fact of one proc body instead of an
invisible adjacency between two procs; Critical/Important tail-branch to
the exported label. Binding class: proc bodies with out(carry:) contracts
through lower_module. Proc params are TYPED in defs (the grammar requires
it): `(d1: u32, d2: u16, d3: u16)` — the register list, clobbers, and
carry-out match the pinned extern contract exactly; the widths are the
twin's documented In sizes.**

### D2 — Init's 32-slot pre-fill (rept site 1)

Comptime fold (`clear_longs` class), displacements DERIVED from the
DMAEntry struct twin:

    comptime fn fill_slot_markers(n: int) -> Code {
        return 0..n |> fold(asm {}, |acc, c| acc ++ asm {
            move.b  d0, {c * sizeof(DMAEntry) + offsetof(DMAEntry, Reg94)}(a0)
            movep.l d1, {c * sizeof(DMAEntry) + offsetof(DMAEntry, Reg93)}(a0)
        })
    }

Slot 0's `move.b` displacement is 0 → the zero-disp collapse to `(a0)`
must match asl (asl collapses; movep is exempt both sides — the frontend
already gates movep out of `collapse_zero_disp`). → **PROBE P6 (parity
against the AS `rept`/`set` twin, miniature n=4). PASSED byte-identical,
including the slot-0 `(a0)` collapse and the movep d16=2 non-collapse.
Binding class: fold-generated Code spliced into a proc body, spliced
comptime-expression displacements.**

### D3 — Process_DMA_Critical jump table (rept site 2) — NAMED-DRAIN-LABEL SHAPE

The twin computes slot targets arithmetically (`bra.w .drain_end-.c*8`).
Label arithmetic in .emp branch-target position is NOT a supported spelling
(Label values participate in arithmetic only in immediate contexts — C1
item 1; `label_arithmetic_is_an_error` pins the data-context rejection),
and hygiene scopes labels per asm-block instantiation, so a fold-generated
hole cannot reference a skeleton label except via a Label ARGUMENT (F2).
Rather than demand a label-arithmetic feature, the .emp names the drain
entry points — byte-identical, and BETTER than the twin's arithmetic (a
cold reader sees which entry drains N):

    .jump_table:
        bra.w   .done                   // slot 0: queue empty
        {jt_filler()}                   // (sizeof(DMAEntry)-4)/2 × trap #0 pad
        {jt_slot(.drain_1)}             // slot 1: 1 entry queued
        ...
        {jt_slot(.drain_7)}             // slot 7
        lea     VDP_CTRL, a5            // slot 8 falls through into the full drain
        lea     DMA_Critical, a1
    .drain_8:
        {dma_send_entry()}
    .drain_7:
        {dma_send_entry()}
        ... (8 groups) ...
    .drain_1:
        {dma_send_entry()}
    .drain_end:  (label kept for the twin's symbol? no — .emp needs no .drain_end)
        move.w  #DMA_Critical, DMA_Critical_Slot
    .done:
        rts

with

    comptime fn jt_slot(drain: Label) -> Code   // lea VDP_CTRL,a5; lea DMA_Critical,a1; bra.w {drain}
    comptime fn jt_filler() -> Code             // trap #0 × (sizeof(DMAEntry)-4)/2
    comptime fn dma_send_entry() -> Code        // move.l (a1)+,(a5) ×3; move.w (a1)+,(a5)

- The `bra.w {drain}` inside jt_slot is a **STRUCTURAL width pin**
  (stride-locked jump-table slot: slot stride must equal
  sizeof(DMAEntry)=14 — lea.l 6 + lea.w 4 + bra.w 4). ONE site comment in
  the fn covers all instantiations. `bra.w .done` in slot 0 is the same
  class (4-byte slot head), site-commented.
- `jmp .jump_table(a1)` stays `jmp` (computed target, idiom list). The
  label rides the d16 field over a1 — legal only because the table sits in
  abs.w-reachable ROM. → **PROBE P4. PASSED byte-identical to asl's
  `jmp label(An)` (base $1000 probe section). Binding class: local label
  as d16(An) displacement in a proc body.**
- Slot count is hand-spelled (8 slots, 8 drain groups):
  `ensure(DMA_CRITICAL_SLOTS == 8, ...)` pins it to the shared const so a
  slot-count change fails loud instead of drifting. The twin keeps its
  rept spelling (lockstep is byte-level, sprites/animate precedent).

### D4 — drain unroll (rept site 3) + Drain_Budgeted_Queue

`{dma_send_entry()}` × 8 covers the Critical drain unroll.
Drain_Budgeted_Queue's `.loop`/`.compact` bodies are RUNTIME loops (the
twin spells each once — no unroll), so they stay literal; the `.compact`
copy targets `(a3)+` (different EA mode), not worth parametrizing the EA
mode into dma_send_entry for one site each.

### D5 — movep / trap / ccr spellings — TWO DEMANDED FEATURES SHIPPED

Probe outcomes (P1-P4 at the real binding class — proc-body instructions,
lower_module pipeline, byte-parity vs sigil-frontend-as):
- P2 `trap #0`, P3 CCR surgery under restored SR, P6 fold pre-fill:
  PASSED as-found (surface existed).
- **P1 movep FAILED as-found — demanded feature 1 (shipped TDD-first):**
  `movep.w Ent.SizeH(a0), d1` mis-fired `[operand.field-overrun]` (.w
  access vs 1-byte field). movep transfers ALTERNATE bytes — each touched
  byte is byte-sized and the bytes past the named field land on OTHER
  fields BY DESIGN (the interleaved layout is the instruction's point).
  Fix: `lower_instr_to_item` maps movep operands with EFFECTIVE access
  width `.b` (the CodeItem keeps the true size). All other movep
  displacement forms (literal / const-name / `{splice}`) passed as-found.
- **P4/P5 FAILED as-found — demanded feature 2 (shipped TDD-first):**
  `jmp .jump_table(a1)` — a local label in d16(An) displacement position
  did not resolve ("unknown name"). Shipped `CodeOperand::DispSymInd`
  (symbolic d16 over An): eval resolves the leading-dot path through the
  label scope BEFORE comptime eval; lowering is a FIXED-width encoding
  with one `Abs16Be` fixup on the ext word (signed-window check = asl's
  constraint), field offset located by the abs-sym sentinel-probe
  technique; loud rejects for data-register base, two symbolic
  displacements, and mixing with a symbolic absolute; the base register
  is reported as a read in flag_check/calls. After both fixes: **all six
  probes PASS byte-identical to the AS front-end**, including P5 (the D3
  named-drain-label table == the AS arithmetic rept table, proving the
  jump-table shape equivalence before the real port).

### D6 — DMAEntry struct twin

`pub struct DMAEntry` joins **engine.structs** (type-only twin module):
field names VERBATIM from structs.asm (Reg94/SizeH/Reg93/SizeL/Reg97/
SrcH/Reg96/SrcM/Reg95/SrcL/Command), sizeof==extern("DMAEntry_len") +
per-field offsetof drift wall (the Act/Sec pattern — structs.asm emits
`DMAEntry_<field>` equs). Consumers: dma_queue.emp now; buffers.asm stays
.asm (no second .emp consumer yet). Single-use derived offsets
(second-slot displacements `sizeof(DMAEntry)+offsetof(...)`) stay
file-local named consts in dma_queue.emp per the structs.emp boundary
rule (one-consumer computed offsets are consumer detail).
First-slot field reads spell `DMAEntry.SizeH(a1)`-style (the `Sst.field`
disp sugar) where the offset is a plain field offset — probed in P1's
struct-sugar arm.

### D9 — QueueDMA contract spelling

    pub proc QueueDMA_Critical (d1, d2, d3) clobbers(d0-d4/a1-a2) out(carry: dropped)

exactly the pinned extern contract (all three entries). The twin header's
"(a3/a5-a6 and d5-d7 preserved)" prose is the license's complement under
the exhaustive-clobbers convention → the .emp header does NOT restate it
as prose (house way: clobbers IS the license); the dplc dead-save reliance
note is kept as a comment pointing at the license, not a disclaimer.
Process_DMA_Critical `clobbers(a1/a5)`; Process_DMA_Important/Deferrable
`clobbers(d0-d1/a0-a3/a5)`; Drain_Budgeted_Queue is jbra'd from both (tail)
— its license `clobbers(d0-d1/a0-a1/a3/a5)` per its header; the callers'
a2 use is their own. Init_DMA_Queue `clobbers(d0-d1/a0)`.

### D11 — load_art.emp structure

`module engine.load_art in load_art`. Procs Art_Decompress (a0/a1 in/out,
version dispatch) + Level_LoadArt (`(a0: *Act)`, the t19 idiom). Externs
with drift-guard comments + kill-list rows: `S4LZ_Decompress (a0, a1)
clobbers(d0-d3/a2-a3) out(a0, a1)` (truth s4lz_decompress.asm:65 — the
no-dict entry touches neither a4/d4), `ZX0_Decompress (a0, a1)
clobbers(d0-d1) out(a0, a1)` (movem-saves a2/d2), `VSync_Wait ()
clobbers(d0)` (game_loop.emp:12 spelling verbatim, row 29 second site).
`use engine.bg` (BG_Init), `use engine.dma_queue` (QueueDMA_Critical —
.emp-owned after file 1, NO extern), `use engine.structs.{Act}`,
`use engine.constants.{ART_*}`.
- Tail calls `bra.w S4LZ_Decompress`/`ZX0_Decompress`: step-2 → `jbra`
  (engine-internal fixed-placement target — the t19 checklist-1 ruling on
  `jsr QueueDMA_Deferrable`; the game_loop cross-section exception does
  not apply). Named decision: **jbra**.
- `.drop_page` stays the out-of-line two-arm shape:
  `if DEBUG == 1 { raise_error "…" } else { jbsr VSync_Wait; jbra .queue_page }`
  with the branch-reach placement comment carried (the twin already
  solved the t19 debug-blob-vs-reach tension out-of-line; byte gates bind
  both shapes).
- `Art_Staging_Buffer`: spelled BARE (house width rule). The twin's
  `lea (Art_Staging_Buffer).l, a1` resolves abs.l — Art_Staging_Buffer =
  Tile_Cache_Nametable sits below the abs.w window in this RAM map, so the
  bare spelling width-selects .l identically. Verified against the
  listing at step 1 (byte gate proves it).
- `move.l #Art_Staging_Buffer, d1` — label-in-immediate idiom (list item).

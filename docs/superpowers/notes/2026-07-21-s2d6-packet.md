## THE CATCH (step-3 correctness): a false dead-save removed (16→15)

`TileCache_WarmupBelowRow d6 bracketing TileCache_DecompressBlock` dropped off the dead-save
worklist. In `DecompressBlock`, **d6 is written ONLY by the `movem.l (a0)+, d0-d6/a2` burst**
(nowhere else — d7 IS written elsewhere, which is why only d6 was affected). The detector
missed that write, so DecompressBlock's computed effect excluded d6, and the dead-save pass
falsely called WarmupBelowRow's `movem.l d6-d7,-(sp)` save **redundant**. A pass-3
register-surgery agent deleting that "dead" save would have **corrupted d6 across
DecompressBlock** — the author's own comment (`// DecompressBlock clobbers d0-d7, needs them
saved`) confirms the clobber is real. A closes the write-set hole → the false dead-save is
gone. This is the completeness fix earning its keep: not theoretical, a live pass-3 footgun
defused.

## R1-tripwire finding: the corpus is NOT literally "0 fresh-pops"

The overseer's strict tripwire ("every `movem (sp)+, X` load has an exact-mask `movem X,-(sp)`
save") FAILS on `Load_Object`: `movem.l d0-d2/a1,-(sp)` … `movem.l (sp)+, d0-d2/a2` deliberately
reloads **a2 from a1's saved slot** (the object template pointer). d0-d2 are pure restores; a2
is a fresh load. Adjudication: a2 IS declared `clobbers(a2)` AND independently written via
`(a2)+`, so the exemption hides no undeclared clobber — sound. The tripwire is implemented as
the **precise** soundness property instead: every `(sp)+`-movem-loaded register is either a
matching save (restore-to-entry) OR declared clobbers/out. Non-vacuous (asserts >0 restores
exist), passes today, fires loudly on a future genuine undeclared fresh-pop.

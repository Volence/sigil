## D1 — GAME_CAMERA_JUMP_LOCK spelling (the engine/game-split seam)

**Decision:** camera.emp takes **`-D GAME_CAMERA_JUMP_LOCK` (0|1)** — the
game_loop.emp precedent verbatim (engine.inc's gate comment documents the same
`-D SOUND_DRIVER_ENABLED / -D SOUND_DEBUG_HOTKEYS` mechanism). The landing-lock
block is `if GAME_CAMERA_JUMP_LOCK == 1 { … }` inside `Camera_Update`, mirroring
the AS twin's `if GAME_CAMERA_JUMP_LOCK` conditional.

Game-symbol references live ONLY inside the gated region:
- `Player_1 + PL_STATE_OFF` (abs.w link-sum — address position, defers fine;
  `equ PL_STATE_OFF = extern("_pl_state")`, the sound_api extern-equ-sum class)
  — the equ itself must be gated with the block (an unconditional module-level
  `extern("_pl_state")` would break a GAME_CAMERA_JUMP_LOCK=0 game's link —
  exactly the property the twin's comment promises away).
- `PSTATE_JUMP`/`PSTATE_ROLLJUMP` sit in `cmpi.b` **imm8** positions — the
  `.b`/`.w` imm-link deferral is UNSHIPPED (kill rows 10/18), so these are
  mirrored comptime consts + `ensure(extern(…) == …)` drift locks, the row-18
  game-mirror class. Mirrors AND their ensures are gated with the block.
- **Binding-class probe owed at step 1** (comptime-if at module scope gating
  consts/equs/ensures — game_loop's precedent gates only INSTRUCTIONS inside a
  proc body; whether top-level items gate is a different binding class and gets
  probed as such). Fallback if module-scope `if` is unsupported: move the consts
  + ensures inside the proc-body gated block (instructions-adjacent), which the
  DEBUG-gated assert precedent (loads inside `if DEBUG == 1`) already exercises.
- **Negative-shape gate artifact:** the port test compiles camera.emp with
  `-D GAME_CAMERA_JUMP_LOCK=0` and WITHOUT the `_pl_state`/`PSTATE_*` equs
  provided, asserting clean lower+link — proving "games without those symbols
  set the flag to 0" as a real property at its real binding class (absence of
  link-time resolution), not a comment. No byte gate for that shape (no
  reference ROM builds it); compile+link-clean is the bar.

## D2 — bg_anim record walk (the 44-byte band record)

Step 1 transcribes the interleaved `(a3)+` walk faithfully (byte gate).
Step 2/4 keeps the WALK (the stream reads are the codegen) but derives every
hand-tracked magic from ONE comptime struct:

    struct bganim_band { driver/rate_shift/step_mask/col_shift/tile_count/
                         vram_dest: u16 ×6, banks: [*u8; 8] }   // 44 bytes

- "banks at a3+6" → `offsetof(bganim_band, banks) - 6` (6 = the three words
  already consumed) as a named const with a site comment;
- `.skip_band addq #6, a3` → named const = sizeof fixed remainder
  (col_shift..vram_dest);
- `.next_band lea 32(a3)` → `sizeof banks` (8×4);
- ensures pin the derived values so the struct can't drift from the walk.

**Twin-scaffolding:** the layout's OTHER author is `tools/inject_editor_bg.py`
(runtime-read act data — no AS-side struct, no extern equs to lock against).
The struct decl adds its kill row SAME COMMIT: review-tier guard (LOCKSTEP
comments both files), kill = the emitter reading its layout from a shared spec,
or table-format change forcing both. The header law — "nothing here may
conditionally assemble on BgAnim_Table's symbols" — survives verbatim: the .emp
references BgAnim_Table only as a link-time address (lea ….l), never comptime.

The 3-word stack juggle around the two DMA queues + `.queue_full addq #6, sp`
unwind is a C2 hazard cluster — step-3(b) contract/stack audit target, and the
dry-panel C2 lens gets pointed at it explicitly.
